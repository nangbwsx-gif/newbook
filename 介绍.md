## 二、数据存储层架构 (Prisma)

### 2.1 数据模型

```
User ──1:N──▶ Book
  │              │
  └──1:N──▶ Category
```

#### User（用户）
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (cuid) | 主键 |
| `username` | String (unique) | 登录名 |
| `password` | String | bcrypt 哈希 |
| `createdAt` | DateTime | 注册时间 |
| `aiSpentMicroCents` | Int (default 0) | 已消耗的 AI 额度（μ¢） |
| `books` | Book[] | 关联的书 |
| `categories` | Category[] | 关联的分类 |

#### Book（书籍）
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (cuid) | 主键 |
| `userId` | String | 所有者（FK → User） |
| `title` | String | 展示名（来自文件名） |
| `fileUrl` | String | 磁盘路径 `/uploads/<uuid>.pdf` |
| `currentPage` | Int (default 1) | 上次阅读进度 |
| `createdAt` | DateTime | 上传时间 |
| `category` | String (default "未分类") | 分类标签（⚠️ 不是外键） |
| `isPublic` | Boolean (default false) | 公开分享开关 |

#### Category（分类）
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (cuid) | 主键 |
| `userId` | String | 所有者（FK → User） |
| `name` | String | 分类名 |
| `createdAt` | DateTime | 创建时间 |
| `@@unique([userId, name])` | 复合唯一 | 同一用户下分类名不重复 |

### 2.2 存储方式

- **数据库：** SQLite（`prisma/dev.db`），适合本地开发，无并发写入压力
- **文件存储：** 本地硬盘 `public/uploads/<uuid>.pdf`
- **架构权衡：** `Book.category` 是字符串而非外键，因为 v2 迭代时 Category 表后加，不想动已有 Book 数据做迁移。代价是每次重命名分类时需要在事务中同步更新 Book 表。

## 三、已落地的业务闭环

### ✅ 3.1 用户认证系统
- **注册/登录：** bcrypt 加密密码，注册即种入 3 个预置分类（医学异常检测、CV综述、多模态大模型）
- **JWT 鉴权：** httpOnly cookie（`auth-token`），7 天有效期，`jose` 库签发验证
- **模块启动守卫：** `JWT_SECRET` 不足 16 字符时抛错（fail-closed）
- **退出登录：** 服务端 `Set-Cookie` 清除，浏览器 JS 无法直接删除 httpOnly cookie
- **"/api/auth/me" 探针：** 首页/阅读页判断当前登录态

### ✅ 3.2 PDF 文件上传与安全存储
- **文件校验：** 后缀名 + MIME 类型 + 魔数字节（`%PDF-` 头）三重校验
- **防猜文件名：** `crypto.randomUUID()` 生成文件名，不再基于时间戳或原名
- **大小限制：** 50MB 上限
- **上传进度：** XHR 监听 `upload.onprogress`（fetch 不支持上传进度）
- **两阶段进度条：** 上传中 → 处理中（写盘 + 写库）
- **DB 失败回滚：** 写库失败时自动删除已落盘文件
- **分类归档：** 上传时可选择目标分类

### ✅ 3.3 PDF 阅读器核心
- **PDF 渲染：** react-pdf + pdf.js worker（本地 `public/pdf.worker.min.mjs`，避免 CDN CORS）
- **翻页：** 键盘快捷键（← →）、热区点击（桌面左右 80px 大区域）、工具栏按钮
- **翻页动画：** CSS 3D 翻书效果（`page-flip-forward/backward`），480ms cubic-bezier
- **缩放：** 0.5× ~ 3.0×，步进 0.25
- **响应式：** `ResizeObserver` 自适应宽度，移动端 `width` 属性替代 `scale`
- **页码跳转：** 输入框 + Enter 提交
- **PDF 目录：** react-pdf Outline 组件，点击跳转
- **日/夜模式：** localStorage 持久化，CSS 变量切换背景色系

### ✅ 3.4 阅读进度同步
- **防抖保存：** `useDebouncedCallback`（1s 防抖），仅文档加载后触发
- **访客隔离：** 公开分享模式下不向 DB 写进度
- **恢复阅读：** 进入阅读器时自动恢复上次阅读页码

### ✅ 3.5 AI 阅读助手
- **DeepSeek 集成：** 通过 `@ai-sdk/openai` 走 `/v1/chat/completions`（非 `/v1/responses`）
- **全文提取：** PDF.js 逐页抽取文本，上限 150,000 字符，带 `[第 N 页]` 标记
- **工具调用：** `navigateAndHighlight` 工具，AI 定位 → 服务端返回入参 → 客户端翻页 + 高亮
- **多步推理：** `stepCountIs(3)` 允许多次工具调用
- **页面控制权在客户端：** AI 只提建议，实际翻页/高亮由 React 监听 `messages[].parts` 执行
- **额度控制：** 每人 1 元（1,000,000 μ¢）免费额度，按 token 计费，超额返回 402
- **访客隔离：** 公开分享模式下隐藏 AI 入口

### ✅ 3.6 关键词高亮
- **模糊匹配：** 忽略空白，逐字符正则匹配
- **碎片阈值：** `max(2, min(4, ceil(keyword.length / 4)))`，避免单字符误伤
- **双层策略：** item 内完整命中 → 精确高亮；子串碎片 ≥ 阈值 → 整体高亮
- **脉冲动画：** 跳转后 1.2s 脉冲动画吸引视觉
- **自动滚动：** 高亮出现后 `scrollIntoView({ block: "center" })`，重试 12 次 × 100ms
- **清除高亮：** 工具栏按钮，双主题配色

### ✅ 3.7 分类管理
- **CRUD 闭环：** 新建 / 重命名 / 删除分类（弹窗内操作）
- **事务安全：** 重命名时同步更新 `Book.category`；删除时将书迁移至「未分类」
- **懒初始化：** 老用户首次调用分类 API 时自动种入 3 个预置分类
- **默认分类：** 注册时自动创建「医学异常检测」「CV综述」「多模态大模型」
- **UI 集成：** Tab 切换 + 数量徽标 + 齿轮按钮进入管理

### ✅ 3.8 公开分享
- **一键开启：** 点击生成链接 → `PATCH isPublic=true` → 复制 URL 到剪贴板
- **一键关闭：** 取消后原链接立即 404（中间件 + 路由双层鉴权）
- **访客模式：** 未登录用户可阅读，但隐藏私人进度、AI 助手、分享入口
- **幂等操作：** 已公开的书再次点击仅复制链接，不发 PATCH

### ✅ 3.9 搜索功能
- **前端实时过滤：** 输入即搜索，大小写无关，书名模糊匹配
- **空状态处理：** 搜索无结果时友好提示 + "查看全部" 按钮

### ✅ 3.10 响应式设计
- **移动端适配：** `matchMedia(768px)` 断点检测
- **自适应 PDF：** `ResizeObserver` 动态计算宽度，移动端用 `width` 替代 `scale`
- **抽屉式侧边栏：** 移动端目录覆盖层 + 遮罩关闭
- **滚动式工具栏：** 按钮多时横向滚动，不换行挤压
- **viewport meta：** `device-width, initialScale=1, maximumScale=5`

## 四、核心路由与目录结构

### 4.1 页面路由 (`app/`)

```
/                    → 产品首页（公开）
/login               → 登录页
/register            → 注册页
/library             → 书橱主页（需登录）
/book/[id]           → 阅读器页（公开分享可访问）
```

### 4.2 API 路由 (`app/api/`)

```
/auth/login          POST    登录
/auth/register       POST    注册
/auth/me             GET     获取当前用户
/auth/logout         POST    退出

/books               GET     获取用户书籍列表
/books               POST    上传 PDF
/books/[id]          GET     获取书籍详情（公开书可匿名）
/books/[id]          PATCH   更新书籍（重命名/进度/分类/公开开关）
/books/[id]          DELETE  删除书籍
/books/[id]/file     GET     流式返回 PDF 字节（公开书可匿名）

/categories          GET     获取用户分类（懒初始化）
/categories          POST    创建分类
/categories/[id]     PATCH   重命名分类（事务同步 Book）
/categories/[id]     DELETE  删除分类（事务迁移 Book 到未分类）

/chat                POST    AI 聊天（流式响应，预算检查 + 扣费）
```
